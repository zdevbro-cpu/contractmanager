# 계약서 PDF-엑셀 매칭 실행안 (파일명 `_` 규칙)

## 1. 적용 범위
- 대상 폴더: `data/` 하위 월별 폴더
- PDF 파일명 규칙: `계약서명_최초수당지급일(YYYYMMDD)_계약자명.pdf`
- 엑셀 원본: `data/★계약현황_월별_검색시트.xlsx`

## 2. 표준 키 정의
- 표준 매칭키(`match_key`)
  - `normalize(계약서명) | YYYY-MM-DD(최초수당지급일) | normalize(계약자명)`
- `normalize` 규칙
  - 앞뒤 공백 제거
  - 연속 공백 1칸으로 축소
  - 비교 시 대소문자 무시(영문 포함 시)

## 3. 파일명 검증 규칙
- 패턴: `^\s*.+_\d{8}_.+\.pdf\s*$`
- 날짜 검증
  - `YYYYMMDD`를 실제 달력 날짜로 파싱
  - 실패 시 `INVALID_DATE`
- 구성요소 검증
  - 계약서명/계약자명 공백 또는 빈값이면 `EMPTY_FIELD`
- 중복 검증
  - 같은 `match_key`의 PDF 2개 이상이면 `DUPLICATE_PDF`

## 4. 엑셀 정규화 규칙
- 필수 컬럼
  - `계약서명`, `최초수당지급일`, `계약자명`
- 날짜 컬럼
  - `YYYY-MM-DD`로 통일 저장
- 금액 컬럼
  - 콤마 제거 후 숫자로 저장, 표시 시 콤마 재적용
- 누락/형식 오류는 `REVIEW_REQUIRED`

## 5. 자동 매칭 우선순위
1. `match_key` 완전 일치
2. `계약자명 + 최초수당지급일` 일치, 계약서명 유사
3. 그 외는 자동 매칭 금지 (`MANUAL_REVIEW`)

## 6. 상태 코드
- `MATCHED`: 자동 매칭 완료
- `UNMATCHED_PDF`: PDF만 있고 엑셀 행 없음
- `UNMATCHED_ROW`: 엑셀 행은 있으나 PDF 없음
- `DUPLICATE_PDF`: 동일 키 PDF 복수
- `INVALID_FILENAME`: 파일명 패턴 불일치
- `INVALID_DATE`: 파일명 날짜 오류
- `REVIEW_REQUIRED`: 사람이 확인 필요

## 7. 저장(반영) 정책
- 1단계: `staging`에 먼저 저장
- 2단계: 검토 완료 후 `confirmed`로 승격
- 반영 단위: 배치(`batch_id`)
- 배치 단위 로그 저장
  - 업로드 시각, 처리건수, 성공/실패/검토건수, 처리자

## 8. 운영 절차
1. `data/` 스캔 후 PDF 메타 수집
2. 파일명 파싱 + 검증
3. 엑셀 정규화 + 검증
4. 자동 매칭 실행
5. `MANUAL_REVIEW` 큐 처리
6. 배치 확정 저장
7. 결과 리포트 출력

## 8-1. 재사용 모듈(일괄등록 공통)
- 공통 라이브러리: `web/src/lib/contractMatching.ts`
- 제공 함수
  - `parsePdfFileName(fileName)`: `_` 파일명 규칙 파싱/검증
  - `buildMatchKey(...)`: 계약서명/최초수당지급일/계약자명 키 생성
  - `matchContracts(rows, pdfFiles)`: 엑셀 행과 PDF 목록 매칭 결과 생성
- 추후 일괄등록에서도 동일 함수 재사용 (규칙 변경 시 한 곳만 수정)

## 9. 수동 검토 기준
- 동명이인
- 동일 날짜 다중 계약
- 계약서명 표기 오타/약어
- PDF 스캔본 품질 문제(파일 열림 불가, 페이지 누락)

## 10. 샘플
- 파일명: `LAS점장점주_20240720_김영수.pdf`
- 파싱 결과
  - 계약서명: `LAS점장점주`
  - 최초수당지급일: `2024-07-20`
  - 계약자명: `김영수`
  - match_key: `LAS점장점주|2024-07-20|김영수`

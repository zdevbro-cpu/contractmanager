# 계약관리 시스템 구현계획

## 기술스택
- Frontend: Vite + React + TypeScript + Tailwind CSS
- Auth: Firebase Auth
- Hosting: Firebase Hosting
- Backend: Firebase Cloud Functions with TypeScript
- Database: Google Cloud SQL PostgreSQL
- File Storage: Firebase Storage

## 보안 원칙
- 주민번호와 계좌번호는 암호화 저장한다.
- 화면에서는 기본 마스킹 처리한다.
- 권한자만 전체정보를 볼 수 있다.
- 계약변경, 파일다운로드, 지급목록 출력은 감사로그에 저장한다.

## 계약변경 원칙
계약정보는 직접 덮어쓰지 않는다.
계좌변경, 양도양수, 증액, 계약해지는 변경요청으로 등록하고 승인 시 반영한다.
변경 전 값과 변경 후 값을 모두 저장한다.

## 수당지급 원칙
수당은 자동계산하되 지급확정 시점의 금액을 별도 저장한다.
지급확정 후 과거 지급내역은 계약정보 변경으로 인해 바뀌면 안 된다.
